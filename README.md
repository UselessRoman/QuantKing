# QuantKing

QuantKing 是运行在个人电脑上的 A 股中长线量化研究与调仓决策工具。它服务于“数据更新 → 研究 → 回测 → 调仓建议 → 人工下单 → 成交登记”的闭环，不以自动下单、实时交易或多用户服务为目标。

> 当前代码为 2.x 实现；本文档及 `docs/` 中标注为 **V3 目标架构** 的内容，是后续重构的准则。未落地的设计不得被描述为当前能力。

## 产品边界

- 市场：A 股，先支持日频数据与周/月度调仓。
- 部署：单用户、本机运行，仅监听 `127.0.0.1`。
- 数据：XTQuant 迁移期可用，AKShare 为免费主接入候选；本地数据仓不依赖任一供应商。
- 交易：系统生成调仓计划，用户通过券商客户端手动下单并登记实际成交。
- 非目标：实时行情、自动下单、高频交易、多租户、云端分布式部署。

## 目标闭环

```text
数据源 → 数据校验/版本化 → 本地数据仓 → 因子与模型研究
     → 回测与评估 → 目标组合/调仓计划 → 人工下单 → 成交登记与组合跟踪
```

## 当前实现概览

现有项目已经包含 SQLite + Parquet 数据仓、XTQuant 数据接入、因子/LightGBM 原型、Backtrader 回测、风险模块和 FastAPI 界面。后续不推倒历史数据和基础测试，而是在相同仓库内按 V3 文档逐步替换边界。

当前本地数据与代码状态应通过以下命令检查：

```powershell
pytest tests -q
python main.py
```

## 开发原则

1. 数据源可替换，业务层不得导入 XTQuant 或 AKShare。
2. 原始数据不可覆盖；统一数据、派生数据与模型产物必须可追溯。
3. 研究、回测和调仓必须使用明确的数据版本与股票池版本。
4. 信号不是订单；所有对外输出均为人工执行的调仓建议。
5. 先建立可复现的 LightGBM 基线，再增加复杂模型或基本面数据。

后续会话和自动化开发工具还必须遵守仓库根目录的 [AGENTS.md](AGENTS.md)。
