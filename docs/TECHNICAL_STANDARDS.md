# 技术规范与验收清单

## 1. 代码边界与目录

目标目录可渐进迁移，不要求一次搬迁：

```text
data/          数据源、标准化、质量、仓储
research/      股票池、因子、标签、实验、模型
backtest/      组合回测和交易规则适配
portfolio/     目标权重、调仓计划、成交、账本
web/           本地 UI、路由、后台任务
core/           领域对象、错误、配置、通用协议
tests/          与源代码目录镜像的测试
```

- 依赖方向只能由外向内：`web → application/domain → ports → adapters`。
- 领域对象使用 `dataclass` 或明确的 Pydantic 模型；禁止在领域层传递无结构 `dict`。
- 供应商 SDK、数据库、qlib、Backtrader 和 FastAPI 都属于适配器层。
- 新公共接口必须含类型标注、docstring、正常返回与失败语义。

## 2. 标准数据契约

Canonical 层使用以下语义：

| 数据 | 主键 | 必填字段/规则 |
|---|---|---|
| instrument | `code` | `code, exchange, name, list_date, status` |
| trading_calendar | `exchange, date` | `date, is_open, previous_open_date` |
| daily_bar | `code, date` | `open, high, low, close, volume, amount`，未复权 |
| adjustment | `code, date, source` | 因子/公司行为及算法版本 |
| index_bar | `code, date` | 与 `daily_bar` 相同的日线语义 |

- Canonical 日期使用 `YYYY-MM-DD`，内部代码标准股票代码为 `000001.SZ`；适配器负责兼容外部格式。
- 价格、成交量和成交额使用 `float64`，不得在数据层无依据四舍五入。
- 单位必须在契约中明确：`volume` 为股，`amount` 为人民币元；供应商单位在标准化前转换。
- 数据帧按 `(code, date)` 升序，主键唯一。

## 3. 数据质量与版本

- Raw 批次不可变；Canonical/Derived 发布新版本，不就地覆写已发布版本。
- 每次发布写入 `dataset_version`、来源、抓取时间、参数、行数、校验和和质量结果。
- 必测规则：schema、单位、主键、交易日、OHLC、缺失/覆盖、异常收益、复权连续性、双源抽样差异。
- 质量失败：保留 Raw，阻止发布，返回结构化错误和报告；禁止返回空 DataFrame 表示成功。
- 读缓存必须在写入或版本切换后失效。

## 4. 研究与模型

- 每个因子必须声明名称、方向、输入列、窗口、复权口径、缺失处理和最早可用日期。
- 因子仅使用当期可见数据；标签仅用于训练，不得进入特征列。
- 所有横截面处理按交易日进行；滚动计算按股票分组进行。
- 默认训练按时间走步切分，不允许随机切分。
- 模型产物必须有 manifest：实验 ID、代码版本、数据/股票池/因子版本、标签、日期段、参数、随机种子、指标和 SHA-256。
- 只加载本地受管理目录中、manifest 校验通过的模型；不得把用户提供的任意路径直接 `pickle.load`。

## 5. 回测与组合

- 回测入口必须显式接收数据版本、股票池版本、信号、组合规则和成本模型。
- 成交规则最少覆盖调仓时点、成交价格、手续费、滑点、涨跌停、停牌和 T+1。
- 研究的复权收益与回测成交价口径分别记录，不得混用。
- 组合构建是唯一逻辑源：回测与调仓页面都调用同一服务。
- `RebalancePlan` 必含计划日期、输入实验、当前/目标权重、订单建议、限制命中项、预计成本和有效期。
- `Fill` 必含人工成交时间、价格、数量、费用和备注；不得伪造券商成交状态。

## 6. Web、任务和配置

- 服务仅绑定 `127.0.0.1`；路由只做输入输出、任务提交和查询。
- 下载、训练、回测必须在后台任务执行，任务状态至少为 `pending/running/succeeded/failed/cancelled`。
- 配置通过环境变量或本地 YAML 读取；机密和本机路径不写入 Git。
- 破坏性操作须二次确认；页面不得直接接受任意文件路径、Python 表达式或模型反序列化输入。

## 7. 测试与质量门槛

| 改动类型 | 最低测试 |
|---|---|
| 数据源/标准化 | 契约测试、单位/字段测试、样本双源对账 |
| 数据仓/版本 | 幂等写入、失败隔离、缓存失效、版本查询 |
| 因子/标签 | 手工可验证样本、无前视、跨实现一致性 |
| 模型 | 时间切分、特征顺序、可复现性、manifest |
| 回测/组合 | 成本、T+1、停牌/涨跌停、调仓计划一致性 |
| Web | 输入校验、任务状态、无券商下单入口 |

跨模块改动必须运行：

```powershell
pytest tests -q -p no:cacheprovider
git diff --check
```

新增依赖前，说明用途、维护状态、许可证、离线可用性和替代方案；不为便利引入云服务或账号绑定。
