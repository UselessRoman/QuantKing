# V3 实施计划与验收标准

## 使用方式

阶段按顺序执行。每个阶段的“完成条件”全部满足后才能开展下一个阶段；小修复可并行，但不得绕开前置数据契约和测试。每项实现任务必须在 PR/提交说明中标注所属阶段与验收证据。

## 阶段 0：基线与工程基础

### 范围

- 备份现有 SQLite、Parquet 和模型文件，登记 XTQuant 历史快照的覆盖范围。
- 固化测试命令，新增格式/静态检查的开发依赖与配置。
- 建立统一错误类型、配置入口和测试夹具的迁移方案。

### 完成条件

- `pytest tests -q -p no:cacheprovider` 通过。
- 已有数据可只读查询，迁移过程不重写历史文件。
- 文档链接、`git diff --check` 通过。

## 阶段 1：数据契约与双源接入

### 范围

- 新增 `MarketDataSource` 协议和标准化器。
- 将 XTQuant 封装为一个适配器；新增 AKShare 适配器。
- 实现证券主数据、交易日历、日线、复权和指数日线的契约。
- 对小样本实现双源抓取与差异报告。

### 禁止事项

- 不得让 AKShare/XTQuant 的 DataFrame 结构直接传入因子或回测。
- 不得让备用源自动覆盖已验证主源数据。

### 完成条件

- 两个适配器通过相同的契约测试和字段/单位测试。
- 日线样本能产出来源、抓取时间、行数、校验和与差异报告。
- 供应商不可用时抛出可识别错误，不返回伪造的空成功结果。

## 阶段 2：版本化本地数据仓

### 范围

- 增加 Raw/Canonical/Derived 目录、版本清单和 SQLite 元数据表。
- 为日线增量写入、复权派生、索引和缓存失效建立明确规则。
- 增加质量检查与失败隔离。

### 完成条件

- 可查询任意 Canonical 数据版本，而不是只查询最新数据。
- 重复入库幂等；一次失败批次不污染已发布版本。
- 质量报告覆盖完整性、主键、OHLC、日期、异常值和双源抽样。

## 阶段 3：可复现研究基线

### 范围

- 建立股票池版本、过滤规则、统一因子定义、标签和走步切分。
- 完成因子评分、Ridge/ElasticNet、LightGBM 三个实验模板。
- 接入 qlib 工作流/信号分析；保留项目拥有的因子与数据语义。
- 新增实验注册表和模型清单。

### 完成条件

- 给定同一数据、股票池、因子、配置和随机种子，结果可复现。
- 训练、验证、测试按日期切分，标签窗口不泄漏。
- 实验产出 IC、Rank IC、ICIR、分期表现、特征重要性和产物清单。

## 阶段 4：组合回测与调仓计划

### 范围

- 实现 `score → target weights → RebalancePlan` 的唯一组合构建服务。
- 支持 Top-K、换手限制、单股/行业上限、现金预留与可交易过滤。
- 统一 qlib 与 Backtrader 的输入与成本假设；定义生产回测口径。
- 新增组合账本、人工成交 `Fill` 和真实/理论净值跟踪。

### 完成条件

- 回测结果可追溯到实验、数据版本、股票池、成本与组合规则。
- 调仓计划可以人工审核，并在登记成交后正确更新组合状态。
- 没有券商连接时，全部流程仍可运行。

## 阶段 5：本地界面与任务化

### 范围

- 数据状态/质量、研究实验、回测、组合、调仓计划和成交登记页面。
- 将下载、因子计算、训练和回测放入本地后台任务。
- 只保留本机同源访问，删除不再需要的自动交易监控页面和认证复杂度。

### 完成条件

- 长任务不会阻塞页面请求，可显示状态、日志摘要和失败原因。
- 用户可从数据版本浏览到实验、回测、调仓计划和成交记录。
- Web 不包含调用券商下单的入口。

## 阶段 6：旧模块退役

### 范围

- 在替代路径经测试验证后，标记或移除自动交易相关模块。
- 删除重复的策略注册和不再使用的 qlib 二进制转换路径。
- 清理过时依赖、文档和测试。

### 完成条件

- 删除前有替代模块、迁移说明和回归测试。
- 依赖清单与实际导入一致；README 不描述不存在的功能。
