# 多因子研究、模型与调仓工作流

实施阶段与完成条件见 [V3 实施计划](V3_IMPLEMENTATION_PLAN.md)，字段、模型产物和回测测试要求见 [技术规范](TECHNICAL_STANDARDS.md)。

## 决策

采用“自有数据与因子语义 + qlib 研究工作流 + LightGBM 主模型”的组合。qlib 用于数据处理协议、实验工作流、信号分析和组合基准；项目自行控制数据版本、股票池、因子定义、成本规则和人工调仓。

不以深度学习、强化学习或自动下单作为首版目标。

## 研究流程

```text
数据版本 + 股票池版本
    → 过滤与复权
    → 因子
    → 标签
    → 训练/验证/测试
    → 分数与 IC 分析
    → 目标组合
    → 回测
    → 调仓计划
```

## 股票池与过滤

研究必须在每个日期使用当时可交易的股票池。首版至少过滤：未上市满指定天数、退市、停牌、ST、价格/成交额过低和不完整历史窗口。历史指数成分股与状态数据未准备好前，不得宣称回测消除了幸存者偏差。

## 因子

首版使用 20–40 个可解释的量价因子：动量、反转、波动率、成交量/流动性和均线偏离。Alpha158 仅作为 qlib 基准对照，不直接视为生产因子集。

所有因子必须：

- 明确输入字段、复权口径、窗口和输出方向。
- 只使用信号日可知的数据。
- 经过截面去极值、标准化和缺失值处理。
- 有 qlib/pandas 一致性测试或唯一计算实现。

## 标签与验证

标签与调仓周期匹配：周度可研究未来 5/10 日收益，月度优先研究未来 20 日相对基准的超额收益。训练、验证、测试按日期严格分割，采用滚动/走步验证；标签窗口不得跨越训练边界。

## 模型路线

| 级别 | 模型 | 目的 |
|---|---|---|
| 基线 1 | 因子分层或 IC 加权评分 | 验证因子本身有效性 |
| 基线 2 | Ridge / ElasticNet | 检验非线性模型是否有增益 |
| 主模型 | LightGBM | 日频表格型多因子预测 |
| 后续 | CatBoost、DoubleEnsemble | 仅在稳定基线上比较 |

每个实验保存数据版本、股票池、因子版本、标签、日期切分、参数、随机种子、模型文件、特征重要性和指标。

## 评价门槛

不以单一收益率决定模型。必须同时报告：

- IC、Rank IC、ICIR 与分年度稳定性。
- Top-K 相对基准收益、最大回撤、夏普/信息比率。
- 换手率、交易成本敏感性、持仓集中度。
- 不同市场阶段的表现。

## 回测与调仓

模型只输出 `date × code → score`。组合模块负责过滤、排名、Top-K、换手约束、单股/行业上限和现金预留，产出目标权重与 `RebalancePlan`。

qlib 的组合回测可作为研究基准；Backtrader 仅用于验证 A 股成交规则。二者必须使用同一份分数、日期、成本和股票池，最终需要指定一套生产口径。

调仓计划包含当前持仓、目标权重、建议股数、参考价、预计成本、风险提示、信号日期与有效期。用户手动下单后，登记 `Fill`（成交日期、价格、数量、备注）以更新本地组合账本。
