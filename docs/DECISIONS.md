# 架构决策记录

本文件记录已确认、会影响后续实现路径的决策。新需求若改变其中任意结论，必须新增或修改相应决策并说明迁移影响；不要以局部代码方便为由绕过决策。

## ADR-001：产品定位

**状态：已接受**

QuantKing 是本机运行的个人 A 股中长线研究与调仓决策工具。主频率为日频，主调仓周期为周度或月度，人工下单。

结果：不建设自动下单、实时账户控制、高频数据、多用户、远程 API 或云端分布式架构。

## ADR-002：数据源不绑定

**状态：已接受**

XTQuant 和 AKShare 通过相同的 `MarketDataSource` 端口接入。供应商返回格式不得泄漏到领域、研究、回测或 Web 层。XTQuant 用于迁移期和历史核验；免费来源失败时通过本地数据版本和显式备用策略保证可用性。

结果：禁止继续把 `data.xt_provider.DataProvider` 当作唯一数据访问入口。

## ADR-003：本地数据仓与版本化

**状态：已接受**

本地 Parquet 是大体量数据的事实存储，SQLite 保存元数据、索引、任务、质量报告和版本清单。数据分为 Raw、Canonical 和 Derived；原始批次只追加不覆盖。

结果：因子、训练、回测和调仓计划必须引用数据版本，而非隐式读取“最新文件”。

## ADR-004：研究方案

**状态：已接受**

采用“自有因子语义与数据版本 + qlib 工作流/分析 + LightGBM 主模型”。先建立因子评分、Ridge/ElasticNet、LightGBM 三个基线，再考虑 CatBoost 或 qlib 的复杂模型。

结果：Alpha158 是对照基准，不是默认生产因子集；首版不做深度学习、强化学习或自动模型搜索。

## ADR-005：统一策略输出

**状态：已接受**

研究模型输出 `date × code → score`。组合层负责可交易过滤、目标权重、换手和集中度约束，并输出 `RebalancePlan`。手工成交写为 `Fill`，用于更新组合账本。

结果：信号不得直接触发订单；回测与实际组合必须共享同一套组合构建规则。

## ADR-006：回测定位

**状态：已接受**

qlib 用于研究基准、信号分析和组合评估；Backtrader 在迁移期用于验证 A 股交易规则。两者不可各自形成独立生产结论，最终生产回测口径必须由同一份分数、股票池、成本和交易规则驱动。

## ADR-007：本地 Web

**状态：已接受**

Web 是本机操作与可视化外壳，业务逻辑在应用服务层。服务仅绑定 `127.0.0.1`，不维护面向公网的 API Key、CORS 或账户交易接口。长任务采用本地后台任务和可查询状态，不阻塞 HTTP 请求。
