# V3 路线图

本文提供方向概览。可执行任务、阶段依赖和验收证据以 [V3 实施计划](V3_IMPLEMENTATION_PLAN.md) 为准；技术细则以 [技术规范](TECHNICAL_STANDARDS.md) 为准。

## 阶段 0：基线保护

- 备份现有 `quant.db`、日线 Parquet 和当前模型产物。
- 为现有 XTQuant 数据标记来源与截止日期。
- 保持现有测试通过；记录迁移前的代表性回测结果。

验收：数据可恢复，`pytest tests -q` 通过。

## 阶段 1：数据源解耦

- 定义 `MarketDataSource` 端口。
- 将 XTQuant 包装为 `XtQuantSource`，新增 `AkShareSource`。
- 建立证券主数据、交易日历、日线、复权和指数的标准契约。
- 实现 Raw/Canonical/Derived 分层、数据版本和质量报告。

验收：同一小样本可从两个源入库；差异可见且不发生静默覆盖。

## 阶段 2：研究基线

- 建立明确的股票池和过滤规则。
- 统一因子计算口径，修复 qlib/pandas 两条路径的不一致。
- 建立因子加权评分、Ridge 和 LightGBM 三个对照实验。
- 使用滚动训练、验证和样本外测试，保存实验记录。

验收：每次训练可复现，并能输出 IC、Rank IC、换手率与样本外表现。

## 阶段 3：组合回测与调仓

- 统一“分数 → 目标权重 → 调仓计划”策略接口。
- 建立月度/周度 Top-K、换手与集中度约束。
- 统一回测成本与交易规则，生成可解释的调仓计划。
- 增加人工成交登记和组合净值跟踪。

验收：一次回测可生成同口径的历史调仓明细；一次实际调仓可登记并反映到组合状态。

## 阶段 4：本地界面

- 数据状态和质量报告。
- 实验、回测和模型对比。
- 当前组合、调仓建议和人工成交登记。
- 长任务后台化并展示进度。

验收：全流程可在本机浏览器完成；不依赖 API Key、外部服务或券商交易连接。

## 暂不建设

- 自动下单与券商账户实时监控。
- 高频、分钟/Tick 策略。
- 多用户、云端部署、微服务。
- 在没有可靠公告日期的情况下使用历史财务因子。
